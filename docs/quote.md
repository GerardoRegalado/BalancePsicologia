## Resumen del proyecto

La cotizacion cubre dos entregables publicos para `www.balancepsicologia.com`:

- Fase 0: pagina temporal en construccion, como entregable rapido previo para mostrar presencia inicial y validar dominio/despliegue.
- Fase 1: landing publica completa para presentar servicios, costos, ubicacion, contacto y solicitud de cita.

La pagina temporal de Fase 0 no reemplaza la landing final. Su objetivo es dar una primera senal visible de avance mientras se trabaja la experiencia completa.

La vision futura de `app.balancepsicologia.com` queda reconocida como una posible fase posterior, pero no esta incluida ni cotizada en este documento.

## Tipo de proyecto detectado

- Tipo: Landing publica / sitio informativo comercial con entregable temporal previo.
- Objetivo principal: Generar confianza, presentar servicios y facilitar solicitudes de cita.
- Nivel de complejidad: Media-baja a media.
- Razon comercial: Aunque no incluye backend complejo, el proyecto requiere presencia inicial rapida, estructura visual cuidada, copy basico, secciones comerciales completas, integraciones ligeras y una presentacion sensible por tratarse de servicios de salud mental.

## Alcance cotizado

### Fase 0 - Pagina temporal en construccion

- Pagina publica temporal en `www.balancepsicologia.com`.
- Mensaje simple tipo "Balance Psicologia esta en construccion" o equivalente profesional.

- Sin formulario.
- Sin agenda funcional.
- Sin captura de datos.

- Secciones aprobadas:
  - Inicio / Home.
  - Nosotros.
  - Servicios.
  - Costos / Pricing / Paquetes.
  - Agenda / solicitud de cita.
  - Testimonios, solo si el cliente entrega testimonios reales y autorizados.
  - FAQ.
  - Contacto.
  - Mapa / ubicacion.
  - Footer con informacion de contacto y enlaces relevantes.
- Boton de WhatsApp.
- Formulario o solicitud de cita basica.
- Envio por correo / SMTP basico, sujeto a proveedor y accesos disponibles.
- Mapa o enlace de ubicacion.

- Google Analytics basico, si el cliente proporciona cuenta o acceso.
- Presentacion de imagenes o PDFs/documentos informativos, si el cliente los proporciona y son necesarios para la landing.
- Preparacion conceptual minima para no bloquear una futura app privada en `app.balancepsicologia.com`.

### Fase 2 futura - No incluida/no cotizada

## Enfoque con IA/Codex

El proyecto se desarrollara con apoyo de IA/Codex para acelerar tareas de implementacion tecnica, organizacion de contenido, refactorizacion, validacion y QA basico.

Esto reduce el tiempo frente a un desarrollo manual tradicional, especialmente en armado de componentes, estructura de secciones, ajustes responsivos y validaciones repetitivas. Aun asi, no elimina el trabajo humano de criterio: direccion visual, decisiones comerciales, revision de copy, integraciones, pruebas, feedback con cliente, ajustes y cierre de entrega.

Por eso esta cotizacion separa dos tiempos:

- Tiempo estimado de implementacion asistida por IA/Codex: horas enfocadas en construccion tecnica.
- Tiempo total estimado del proyecto: calendario realista que incluye diseno, validaciones, feedback, QA, ajustes y coordinacion.

## Supuestos

- La cotizacion cubre solo Fase 0 y Fase 1.
- La Fase 0 es una pagina temporal simple y no reemplaza la landing final.
- La agenda de Fase 1 se implementa como solicitud de cita o contacto inicial, no como sistema automatico de calendario.
- Los testimonios se integran solo si el cliente entrega contenido real, autorizado y apto para publicacion.
- Si no hay testimonios disponibles, la seccion puede quedar preparada, oculta o pendiente sin inventar contenido.
- Google Analytics queda incluido solo si existe acceso o cuenta disponible.
- El envio por correo / SMTP queda sujeto al proveedor elegido y a que se proporcionen accesos correctos.
- La fase futura de app privada requiere una cotizacion independiente.

## Riesgos y ambiguedad

- Si no hay acceso al dominio o hosting, la Fase 0 puede retrasarse aunque sea un entregable simple.
- Si el cliente no entrega contenido final, la Fase 1 puede avanzar con copy preliminar, pero la publicacion final puede retrasarse por aprobaciones.
- Si faltan testimonios reales, esa seccion no deberia publicarse con contenido ficticio.
- Si no hay referencias visuales ni material fotografico, se trabajara con una propuesta inicial basada en los atributos definidos: calido, humano, confiable, limpio, profesional y tranquilo.
- Si se solicita almacenar datos sensibles de pacientes, el alcance cambia y debe tratarse como fase privada con seguridad, privacidad, permisos y base de datos.

## Vision futura no cotizada

`app.balancepsicologia.com` queda como fase futura no incluida en esta cotizacion.

Esa fase podria contemplar login, base de datos, pacientes, historial clinico, notas, finanzas, panel administrativo, permisos y manejo seguro de informacion sensible. Por su naturaleza, debe cotizarse y definirse por separado con criterios de seguridad y privacidad antes de cualquier implementacion real.
